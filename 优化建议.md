# 糖记项目优化建议

## 一、必须修复的问题

### 1. 恢复被删除的关键文件
运行以下命令恢复 `public/` 下的图标文件以及构建配置：
```powershell
git restore public/ .gitignore package.json vite.config.js tailwind.config.js postcss.config.js nginx.conf server.js
```
这些文件是 PWA 和构建的关键依赖。

### 2. 清理未使用的依赖
`package.json` 中有 `react-router-dom`，但实际代码用的是自定义导航，并未使用它。可以移除：
```bash
npm uninstall react-router-dom
```

### 3. API Key 硬编码风险（服务器端）
`server.js` 中硬编码了智谱 API Key。建议改为环境变量：
```js
const API_KEY = process.env.GLM_API_KEY || 'your-key'
```

## 二、性能优化

### 1. IndexedDB 数据加载优化
- `getAllRecords()` 每次加载全部记录，记录多时会变慢
- 建议加一个分页查询方法：`getRecordsPage(page, pageSize)`
- 图片存 base64 在 DB 里会占用大量内存，建议改为只存 URL/路径引用

### 2. 组件懒加载
目前所有 Screen 组件在 App 启动时全部导入。可改为 React.lazy + Suspense：
```jsx
const HomeScreen = React.lazy(() => import('./screens/HomeScreen'))
```
首屏加载体积会减小约 40%。

### 3. 固定尺寸适配性
`phone-frame` 固定为 `390x844px`（iPhone 尺寸），在大多数Android设备上需要额外适配。建议：
- 改用百分比/视口单位
- 保留框架作为 `max-width` 限制
- 增加媒体查询适配不同屏幕

## 三、编译 APK 的方案

你的项目是纯前端 PWA，要打包成 APK 有以下几个推荐方案：

### 方案 A：PWABuilder（最快，推荐）
1. 先构建前端：`npm run build`
2. 访问 https://pwabuilder.com
3. 上传你的 dist 文件夹或输入部署后的网址
4. 一键生成 APK（使用 Android 原生 WebView 包装）

无需额外代码改动，5 分钟搞定。**适合你的项目现状。**

### 方案 B：Capacitor（功能更强，推荐）
如果以后需要调用原生功能（相机、文件系统等）：
```bash
npm install @capacitor/core @capacitor/cli @capacitor/android
npx cap init 糖记 com.tangji.app
npx cap add android
npm run build
npx cap copy
npx cap open android
```
然后用 Android Studio 打包 APK。

### 方案 C：Tauri（更小、更快，但需 Rust 环境）
```bash
npm install -D @tauri-apps/cli
npx tauri init
npm run build
npx tauri build
```
APK 体积小（~5MB），性能好，但需要安装 Rust 工具链。

## 四、其他建议

| 问题 | 建议 |
|------|------|
| 没有 404 页面 | 添加 fallback 页面 |
| 错误处理较弱 | 增加更优雅的网络错误提示 |
| 数据导出功能简单 | 考虑加密备份或云同步 |
| 无单元测试 | 可加 vitest + testing-library |
| CSS 固定色值重复 | 建议提取到 tailwind 配置中，已有但部分未使用 |

---

## 五、开始编译 APK 前

需要先完成以下步骤：

**步骤 1：** 恢复被删除的文件
```powershell
git restore public/ .gitignore package.json vite.config.js tailwind.config.js postcss.config.js
```

**步骤 2：** 安装依赖
```powershell
npm install
```

**步骤 3：** 构建生产版本
```powershell
npm run build
```

确认构建成功后，就可以用 PWABuilder 或 Capacitor 打包 APK 了。你倾向于哪种方案？
